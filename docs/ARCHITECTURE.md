# PalusOS architecture

PalusOS is intentionally packaged as a thin, understandable product shell around a deeper trading research system.

## Modules

```text
market data adapter
  ↓
strategy / agent runner
  ↓
replay + paper evaluation
  ↓
execution calibration
  ↓
robustness and safety gates
  ↓
human-readable truth report
```

## 1. Market data adapter

Initial adapter: PumpFun launch markets using Solana/PumpPortal style flow.

Future adapters:

- Solana DEX thin-liquidity pairs;
- perps or spot markets;
- third-party agent decision logs.

## 2. Strategy runner

Runs either:

- candidate profile JSON generated by a quant/ML search process;
- a third-party autonomous agent's proposed decisions;
- a baseline strategy used for comparison.

## 3. Evaluation engine

Computes:

- paper PnL;
- quote-adjusted EV;
- win/loss distribution;
- largest-winner-removed robustness;
- loss cluster/drawdown;
- rug/gap-loss exposure;
- proof density.

## 4. Promotion gate

Outputs one of three decisions:

- **Kill** — rejected before live funds.
- **Keep Paper** — promising but needs more proof.
- **Canary Eligible** — may graduate to tiny guarded dev-wallet canary.

## 5. Report UI

The UI turns machine evidence into a judge/builder-readable audit trail:

- what the agent tried;
- why paper PnL is or is not believable;
- what the execution-adjusted result says;
- what safety gates passed or failed.

## Public-safe boundaries

This repo excludes secrets, private databases, `.env` files, private wallets, and any scaled-live trading configuration.
